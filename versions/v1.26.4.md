# v1.26.4

## Reset do Auth Lobby para mundo próprio

O Auth Lobby deixou de usar o mapa externo disponibilizado anteriormente.

Motivo:

- o mapa não possui permissão segura de uso;
- manter qualquer parte dele poderia gerar risco futuro;
- a solução correta é um `minecraft:overworld` vazio, próprio do Atlas.

## Operação realizada

- Criado backup antes do reset:

```text
/opt/atlas/server/backups/worlds/auth-overworld-external-map-removal-20260705-181028
```

- Removidos os dados físicos antigos do Auth Lobby:

```text
world/region
world/entities
world/poi
```

- Preservados:

```text
playerdata
stats
advancements
cobblemonplayerdata
cobbledollarsplayerdata
datapacks
dimensions/atlas
serverconfig
```

## Mundo void

Adicionado datapack próprio:

```text
world/datapacks/atlas-auth-void
```

Ele redefine o `minecraft:overworld` como:

```text
type: minecraft:flat
biome: minecraft:the_void
layers: []
features: false
lakes: false
```

Também foi ajustado no `server.properties`:

```text
level-type=minecraft:flat
generator-settings={"layers":[],"biome":"minecraft:the_void","features":false,"lakes":false}
```

Características:
