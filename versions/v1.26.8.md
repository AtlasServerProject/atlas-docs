# v1.26.8

## NPC visual migrado para Easy NPC

O NPC visual do Auth Lobby deixou de ser criado diretamente pelo Atlas Core.

Motivo:

- `Villager` ficava visualmente fraco para o padrão do lobby;
- `FakePlayer` nativo do Fabric existia no servidor, mas não renderizava corretamente para o cliente;
- `ArmorStand` funcionava como fallback, porém ficou visualmente ruim;
- a solução correta é usar um mod de NPC compatível com Fabric.

## FancyNPCs

Foi avaliado o FancyNPCs como referência visual, porém ele é voltado para Paper/Folia/Bukkit e não carrega em servidor Fabric.

Como alternativa Fabric, foi instalado o Easy NPC.

## Mods instalados no servidor

Versão escolhida:

```text
Easy NPC: Core 6.0.21
Easy NPC: Config UI 6.0.21
Minecraft: 1.21.1
Loader: Fabric
```

Arquivos:

```text
/opt/atlas/server/fabric/mods/easy_npc-fabric-1.21.1-6.0.21.jar
/opt/atlas/server/fabric/mods/easy_npc_config_ui-fabric-1.21.1-6.0.21.jar
```

Observação:

- a versão `6.25.0` foi testada, mas exige Fabric Loader `0.18.3+`;
- o servidor Atlas usa Fabric Loader `0.17.3`;
- por isso foi usada a versão `6.0.21`, compatível com `>=0.16.4`.

## Mudança no Atlas Core

O `LobbyNpcService` agora não cria mais corpo visual de NPC.

Ele mantém apenas o comportamento:

- qualquer entidade no Auth Lobby com a tag `atlas_npc_auth_emerald`;
- ao ser clicada por um jogador autenticado;
- envia o jogador para o Lobby Emerald.

## NPC criado no mundo

Foi criado um NPC Easy NPC do tipo:

```text
easy_npc:humanoid
```

Posição:

```text
X: 628.622
Y: 123.0
Z: 3535.462
Yaw: -90
```

Tag Atlas:

```text
atlas_npc_auth_emerald
```

Nome:

```text
Lobby Emerald
```

## Comando usado

```mcfunction
execute in minecraft:overworld run summon easy_npc:humanoid 628.622 123 3535.462 {Tags:["atlas_npc_auth_emerald"],CustomName:'{"text":"Lobby Emerald","color":"green"}',CustomNameVisible:1b,Invulnerable:1b,NoAI:1b,NoGravity:1b,PersistenceRequired:1b,Rotation:[-90f,0f]}
```

## Validação

- Servidor iniciou com Easy NPC carregado.
- Datapacks `easy_npc` e `easy_npc_config_ui` foram detectados.
- Comandos `/easy_npc` e `/easy_npc_config_ui` foram registrados.
- NPC `easy_npc:humanoid` foi criado no ponto correto.
- TPS após estabilização: `20.00`.

## Cliente

Easy NPC é `client required`.

Quem for entrar no servidor precisa ter no modpack:

```text
easy_npc-fabric-1.21.1-6.0.21.jar
easy_npc_config_ui-fabric-1.21.1-6.0.21.jar
```
